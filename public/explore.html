Explore Our API

Discover how our Spotify API works and what you can build with it. Perfect for developers and music enthusiasts.

API Endpoint

GET /api/spotify?url={spotify_url}

Main endpoint to fetch track information and download links from Spotify URLs.

Request Parameters

Parameter Type Required Description
url string Yes Valid Spotify track URL

Response Format

{
  "success": true,
  "data": {
    "title": "Artist - Song Name",
    "artist": "Artist Name",
    "name": "Song Name",
    "duration": "3:45",
    "popularity": "85%",
    "preview": "https://preview.url",
    "thumbnail": "https://image.url",
    "url": "https://spotify.url",
    "downloadLink": "https://download.url"
  }
}

Example Usage

JavaScript (Fetch)

const response = await fetch('/api/spotify?url=https://open.spotify.com/track/example');
const data = await response.json();

if (data.success) {
    console.log(data.data.title);
    console.log(data.data.downloadLink);
}

cURL

curl "https://your-domain.vercel.app/api/spotify?url=https://open.spotify.com/track/example"

Python

import requests

url = "https://your-domain.vercel.app/api/spotify"
params = {"url": "https://open.spotify.com/track/example"}

response = requests.get(url, params=params)
data = response.json()

if data["success"]:
    print(data["data"]["title"])
    print(data["data"]["downloadLink"])

Error Handling

The API returns appropriate HTTP status codes and error messages:

Status Code Description Response
400 Bad Request {"success": false, "error": "URL parameter is required"}
500 Internal Server Error {"success": false, "error": "Track not found"}

Rate Limits

Please be respectful with API usage. While there are no strict rate limits, excessive requests may be throttled to ensure service availability for all users.
